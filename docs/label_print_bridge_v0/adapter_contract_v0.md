# 印刷ブリッジ アダプタ契約 v0（ラベル印刷ブリッジ v0.4）

内製コアと外注ブリッジ本体の**境界仕様**。コアは抽象「印刷ブリッジ」へ**ラベルペイロード**を渡すだけ。
b-PAC / Brother TD-2350 の具体はブリッジ側に閉じ込める（**機種抽象化**／11.4 アダプタ層で差替可能）。

## 1. 役割分担

| 区分 | 担当 | 内容 |
| --- | --- | --- |
| コア（内製） | 自社 | ラベルペイロード生成・PDF出力・印刷ON/OFF＋送信フック・履歴/再印刷・バーコード枠 |
| ブリッジ（外注・孤立部品） | スポット外注 | ローカルWindows agent：ペイロード受信 → b-PAC → Brother TD-2350 印刷。`.lbx` 差込 |

境界は**この契約のみ**。コアはブリッジの実装（b-PAC・.lbx・機種）を知らない。

## 2. ラベルペイロード（機種非依存・入力契約）

1荷物 = 1ラベル。**数字のみ**（住所・氏名は含めない＝現行どおり）。

```jsonc
{
  "items": [
    {
      "tracking_number": "9000000000012", // 小ラベル＝問合番号（数字のみ）
      "basket_code": "A",                  // 大ラベル前半＝かご記号
      "delivery_order": 1,                 // 大ラベル後半＝配達順
      "office_code": "A01",                // 任意（履歴・振分用）
      "kind": "print",                     // print | reprint | pdf
      "terminal_id": "T-001"               // 任意（端末別ON/OFF識別）
    }
  ]
}
```

- 大ラベル表示 = `basket_code + " " + delivery_order`（例「西-5 1」。本DBは「A 1」「M01 2」等）。
- 小ラベル表示 = `tracking_number`。
- ラベル様式（サイズ・配置）は `.lbx`（外注）側で確定：**大サイズ36＝かご記号＋配達順／小サイズ12＝問合番号／高さ約30mm**。残る入力は**用紙幅**と正確な配置のみ。

### バーコード枠（既定OFF）

- `tracking_number` をバーコードで描ける**枠のみ**を `.lbx`／PDF に用意。**既定は非表示（OFF）**。
- 配送伝票（荷物に付く伝票）に問合番号バーコードがあるため通常は刷らない（§8.5 照合・仕分けスキャンは**配送伝票**を読む）。
- 将来ONにする時のみ、**シンボロジーを配送伝票に合わせる**（それまで確定不要）。

## 3. トランスポート（入出力契約）

ブリッジは次のいずれかで受ける（外注が選択／複数可）。コアは**差替できる**よう URL/パスを設定値で持つ。

| 方式 | 受け口 | 備考（推奨度） |
| --- | --- | --- |
| ローカルHTTP | `POST http://localhost:<port>/print`（body＝上記JSON） | ◎ 推奨。コアは設定の「ブリッジURL」へPOST |
| ファイル投下 | 監視フォルダへ `*.json`（上記JSON）を書き出し | ○ HTTP不可環境向け |
| キュー | ローカルキュー（任意） | △ 大量時 |

- **出力＝印刷**（Brother TD-2350）。コアへの戻りは `200 OK`（受理）で十分。実印刷の成否はブリッジのログ/UI側。
- b-PAC は **Windows COM（ローカル実行）＝クラウド不可**。だからブリッジは**ローカルWindows agent**（孤立部品）。

## 4. 印刷ON/OFF＋送信フック（コア側）

- **ON** … 仕分け確定/操作時に送信フックが発火し、ブリッジへ上記JSONをPOST。**端末別**（localStorage）。
- **OFF** … 送信しない（**読取のみ**）。
- ブリッジ**未接続でも**、コアは PDF出力・履歴記録で運用継続（フォールバック）。

## 5. 差替性（機種抽象化の確認）

- コアが渡すのは**ペイロード（§2）＋トランスポート（§3）**だけ。
- ブリッジを別実装（別プリンタ・別ライブラリ）に差し替えても、**この契約を満たせばコア無改修**。
- ＝ 11.4「印刷はアダプタ層で差替可能」を満たす。
