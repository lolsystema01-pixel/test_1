### 実機確認の前提（★注意）

現在の検証DBは **IT01 に** `https://drive.google.com/drive/folders/xxxxxxxx`**（プレースホルダ）が入っています**（出力の保存先 v0.3 の検討中に投入されたもの）。
そのままだと **IT01 は「完了」扱いで初回ゲートが出ません**。実機確認では次のいずれかを行うこと。

```sql
-- (a) IT01 で初回ゲートを確認したい場合：未完に戻す
update public.offices set gdrive_folder_url = null where office_code = 'IT01';

-- (b) 別の営業所（A01/C01 は NULL のまま）で確認する場合は何もしなくてよい
```

※ プレースホルダのまま運用すると「完了しているのに保存先が存在しない」偽の完了になるため、
　 いずれにせよ実URLに置き換えるか NULL に戻すこと。

## 申し送り

- **権限の設計は業務Aの確認事項**（README参照）。指示書が「初回ログインユーザーとの整合は要確認」と
保留していた点について、本実装は「hq＝常時／area＝自営業所かつ初回のみ」の折衷を採った。
hq 限定にすると画面は出るが保存できないデッドロックになるため。
- **消費側（Gドライブ保存）は未実装**。この列は保存先の指定を持つだけで、実際にDriveへ書く仕組みは
出力の保存先 v0.3 の担当だが、**Drive 認可基盤が存在せず方式選定で保留中**（(b) OAuth＋drive.file 推奨・
(c) Workspace＋サービスアカウントは経営判断）。→ **URLを入れてもまだDriveには保存されない。**
- `offices` に `B01`（営業所B）が残っている。ダミーデータ正準規格v1では「旧B01は廃止」のはずで、
取り残しの可能性がある（本モジュールとは無関係・別途確認）。
