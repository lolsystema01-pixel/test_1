# 確認結果メモ：エリアマスタ文字化け修正（語彙是正① 単独版）v0.1

対象：`area_master` / `area_master_staging` の U+FFFD（chr(65533)）破損を、原本CSV `エリアマスタ_最終3_nobom.csv`（84,023行・U+FFFD 0件）と intact列で突合して修正。

## ①-a 特定
| テーブル | 実測 | 引き継ぎ期待 |
| --- | --- | --- |
| area_master_staging | **43** | ≈20 |
| area_master | **37** | ≈13 |

期待の2〜3倍だったため指示書どおり**一旦停止し、破損の“質”を診断**（`diagnose_mojibake_v0.sql`）。以下を確認し、質は同じ（＝individual突合で続行可）と判断：

- **staging 43行はすべて「1行1フィールドのみ破損」**。common_id は **43行すべて intact**（突合キーに使える）。
  破損列内訳：src_town_key=20 / is_valid=6 / municipality=5 / town=5 / depot=3 / prefecture=3 / area=1。
- **area_master 37 = staging 43 − 6**。6 は **is_valid が化けた行**で、load の「有効のみ」取込で除外されていた（→修正後の再loadで復活）。
- area_master の **2列破損=13行 = town_key破損13行**。`town_key = normalize(pref+muni+town)` の派生列に、住所セル破損（pref3+muni5+town5=13）が写り込んだもの。**新種の破損ではない**。common_id は37行すべて intact。

## ①-b / ①-c 修正
成果物 `fix_area_master_mojibake_v0.sql`（全UPDATE/DELETEは同ファイルに列挙。ctidは各行コメントに併記）。

- **①-b**：staging の破損43セルを**原本CSVの該当列の値**へ個別UPDATE。
  WHERE は **intact列（原本CSVで一意を実証済）＋「対象列がU+FFFDを含む」ガード**。ctid非依存で、キー不一致なら0件更新＝健全行は不変。
- **①-c**：area_master の破損行を「U+FFFDを含む行」条件で DELETE（37行）→ `area_master_v0/area_master_load_v0.sql` を再実行して再生成。
- **禁止事項の順守**：src_town_key の中身をパースした自己修復は**していない**（原本CSVの列値を権威に採用）。単一列JOINの一括UPDATEも**していない**。
- **突合結果**：43行すべて一意確定（**FLAG 0**）。
- **事前検証（pglite）**：破損を再現→生成UPDATE適用で「U+FFFD 0件・値一致・オトリ健全行不変・破損行のみDELETE」＝**9/9 PASS**。
