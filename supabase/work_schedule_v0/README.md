# ドライバー参照＋稼働予定 v0（申請・承認の土台）

指示書 `shijisho/shijisho_work_schedule_v0_1.docx` の成果物。
ドライバー（マスタ）を参照し、稼働予定の **申請→承認/却下**・**営業所直接入力**・**申請可能期間**・
**予測対象日の稼働人数** までをSQLで回す。T3アプリUIは後続。
要件定義 6.4 / 9.2 / 5.3 / 6.5 に対応。

## 前提

- DBスキーマ v0（`drivers` / `work_schedules`）作成済み。機微テーブルのRLSは RLS v0.2 で設定済み。
- 拠点振分 v0.2 実行済み（offices = `A01`愛知県1営業所 / `C01`愛知県2営業所）。
- 当日基準は `current_date`（日付は相対指定なので実行日に依存しない）。

## 実行順（SQL Editor に手でコピペ）

1. `seed_drivers_v0.sql` … ドライバーマスタの少数ダミー（A01に2名・C01に2名）。
2. `work_schedule_v0.sql` … §1 申請（期間チェック）→ §2 承認/却下 → §3 営業所直接入力 → §4 期間チェック確認。
3. `headcount_v0.sql` … 予測対象日の稼働人数（営業所×日付・承認済み・0人も0）。
4. `check_schedule_v0.sql` … 状態遷移・フォールバック・期間・人数の確認。

各ブロックを選択して Ctrl/Cmd+Enter で個別実行。

## 状態遷移

```
申請（ドライバー）   →  application_status = 申請中
   │  営業所が更新
   ├─ 承認 → 申請中→承認（転記なし・同じ行が承認状態）
   └─ 却下 → 申請中→却下
営業所直接入力（フォールバック）→ 承認で直接 insert（アプリ未使用者・訂正用）
申請可能期間: current_date ≤ 稼働日 ≤ current_date + offices.request_period_days（範囲外は弾く）
稼働人数: application_status='承認' を 営業所×日付 で count（0人も0と表示）→ 配車(6.5)の入力
```

- 配車本体（6.5）。スキルの自動算出・調整UI。本番データ・現行GAS。SQLの自動実行・マイグレーション。
