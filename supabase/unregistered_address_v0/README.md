# 未登録住所の記録・修正フロー v0

指示書 `shijisho/shijisho_unregistered_address_v0_1.docx` の成果物。
判定不能だった住所を**記録 → 一覧 → 修正 → 再判定**で解消し、住所確認中（保留）の荷物を取りこぼさず管理する。
要件定義 6.3 前半（未登録住所の一覧・修正・エリア再判定）/ 9.1（未登録住所）/ 6.2（再判定）に対応。

## 前提

- 住所判定 v0.2（`supabase/address_match_v0/`）と拠点振分 v0.2（`supabase/office_assign_v0/`）が実行済み。
- `public.unregistered_addresses` は normalize_v0.sql ② で作成済み（無くても本SQL §1 が作る）。
  - 列（9.1）: `tracking_number` / `address` / `normalized_address` / `reason`(対応区分) / `resolved`(対応済み) / `note`(メモ)。
- 初期状態: 判定不能2件（名古屋市栄／存在しない町）が記録・保留済み。

## 実行順（SQL Editor に手でコピペ）

1. `unregistered_v0.sql`
   - §1 テーブル（無ければ作成）／§2 記録（再同期・冪等）／§3 **一覧**（未対応）
   - 【人】§4 **修正**: §3で見た問合番号と正しい住所を書き込んで実行
   - §5 **再判定**（住所→共通ID→拠点→営業所を保留荷物に再適用）／§6 件数
2. `check_unregistered_v0.sql` … 未登録件数・解消件数・残保留件数・取りこぼし無しの確認。

各ブロックを選択して Ctrl/Cmd+Enter で個別実行。

## フロー

```
判定不能（共通ID無し）─→ unregistered_addresses に記録（resolved=false）＋荷物=保留
        │
        ▼ §3 一覧（未対応）
   【人】§4 住所を修正（正しい住所／対応区分）
        │
        ▼ §5 再判定（match_v0 + assign_office_v0 と同一ロジックを保留荷物に再適用）
   ├─ 解消: 共通ID・拠点・営業所を付与 → status=未配車 / resolved=true
   └─ 未解消: 保留のまま（取りこぼさない。例: Master未収録の名古屋市）
```

## やらないこと（範囲外）

- マスタ整合性検査・修復（6.3後半: 拠点コード不一致・TownKey重複検査等）。別指示書。
- 未登録住所の修正UI（フロントは後続。v0はSQLベース）。住所正規化ルール本体の拡張。
- 配車（6.5）・仕分け・帳票。本番データ・現行GAS。SQLの自動実行・マイグレーション。RLS詳細（別指示書）。
