## 備考・気づき

- **読み取りは検証Supabaseに実接続**（lookup.ts→`delivery_status_public`・anon・非PII）。env未設定はダミーへフォールバック。受付登録・OTP送信は実接続せずダミー（本番DB/鍵には触れない・検証DBのanonキーのみ）。実接続テストは実在の半角英数番号（例 900000000005）で。
- スマホ実機は `npm run dev -- --host` → 同一WiFiで `http://<PCのIP>:5173/`（FWでNode受信許可が必要）。
- 認証中核（N-3）はアプリ内＝外注に出さない。トークン必須で番号のなりすまし防止。受付/状態は body の番号でなくトークンから解決。
- PII：状態取得は市レベルのみ・氏名/連絡先/詳細住所は返さない。ログは mask.ts で伏せる（N-10）。
- バリデーションの「正」は validation.ts（N-2/D章）。サーバ（N-4）でも validateAll を再実行＝多層防御。
- 範囲外：LINE/SMS/電話（N-7〜N-9）・完了通知メール文面・本番接続。
