## 備考・気づき

- **service_role キーをアプリに置かない設計**。状況確認も取込も anonキー＋ログイン荷主の JWT で動く。
取込の書込みは DB の SECURITY DEFINER 関数 `shipper_import_deliveries` だけが行い、`shipper_id` を `my_shipper()` に固定（自社のみ）。
- `deliveries` の RLS は **SELECT 専用のまま**（INSERTポリシー＝書込みRLSは足していない）。書けるのは関数経由のみ。
- 別 Cloud Run は作らず、将来移設可能な形。「API連携取込（荷主システム直結）」は将来＝範囲外。
- `deliveries.recipient_name` 列は荷主マスタ取込で付与済みが前提（未実行だと取込関数が列無しエラー）。
